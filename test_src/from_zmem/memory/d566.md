# CLI and build_runner must share pvtro.yaml, always generate the canonical lib/pvtro.g.dart artifact, and copy to a configured output path when it differs.

## Context

The package supports both direct CLI execution and build_runner integration. The repository's validated examples depend on both modes honoring the same root configuration file.

## Content

`pvtro.yaml` is the shared config source.

`lib/pvtro.g.dart` is always the canonical generated artifact.

When `output` differs from the default, PVTRO writes identical generated content to the configured path as well.

## Consequences / notes

Documentation and tests must treat canonical-plus-copy behavior as the contract for custom outputs.
