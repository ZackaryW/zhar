# PVTRO is consumed from the root app of a multi-package Flutter project and generates a TranslationProvider wrapper plus pvtroSyncPackageLocales for every discovered slang package in the dependency graph.

## Context

The consuming app is expected to depend on one or more internal packages that also use slang and expose generated translation files under `lib/`.

## Content

Users add PVTRO as a dev dependency, run it from the root app, and then import the generated root integration file into the application bootstrap path.

The examples in `example/` model the expected workflow with nested path dependencies.

## Consequences / notes

The root app is the only place where PVTRO writes outputs; nested packages remain source inputs, not generation targets for this tool.
